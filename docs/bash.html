Cuando empecé a utilizar linux y más concretamente ubuntu algo con lo que me empecé a usar fue la consola. Al principio no le veía mucha utilidad, pero con el tiempo y conociendo como funcionaban las tripas del sistema operativo el obligatorio saber utilizarlo.

La potencia que obtienes y las opciones que te proporciona, te permiten que casi cualquier cosa que te imagines la puedas implementar para que linux se adapte a lo que necesitas en cada momento.

Para hacer un poco de historia, Bash es el intérprete de comandos de Unix, escrito para el proyecto GNU. El nombre de Bash viene del acrónimo de bourne-again shell (otro shell bourne) - haciendo un juego de palabras (born-again significa renacimiento) sobre el Bourne shell (sh), fue uno de los primeros shells importantes de Unix.

Hacia 1978 el shell Bourne era el shell distribuido con el Unix Version 7. Stephen Bourne, por entonces investigador de los Laboratorios Bell, escribió el shell Bourne original . Brian Fox escribió el shell bash en 1987.

En 1990, Chet Ramey se convirtió en su principal desarrollador. bash es el shell por defecto en la mayoría de sistemas GNU/Linux, además de Mac OS X Tiger, y puede ejecutarse en la mayoría de los sistemas operativos tipo Unix. También se ha portado a Microsoft Windows por el proyecto Cygwin, a pesar de ser nativos para las plataformas Unix.
